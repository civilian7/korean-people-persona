# LLM 에이전트 활용 사례

korean-people-persona MCP 서버를 LLM 에이전트에서 활용하는 대표 사례 4가지를 정리한다.
모든 사례는 동일한 공통 파이프라인의 변형이다.

## 공통 파이프라인

```
aggregate (모집단 분포 파악)
  → sample_persona / search_persona (표본 추출)
    → get_persona (페르소나 전체 컨텍스트 로드)
      → LLM 롤플레이 (페르소나에 빙의해 과제 수행)
        → 결과 집계
```

이 데이터셋의 차별점은 **표본이 실제 한국 인구 분포에 근접한 통계 기반**이라는 것이다.
"30대 직장인을 상상해줘"가 아니라, 시도×성별×연령 분포 그대로 1,000,000명에서
샘플링하므로 결과에 분포 근거가 생긴다.

도구 5종 요약:

| 도구 | 역할 |
|------|------|
| `stats` | 전체 분포 개요, 사용 가능 컬럼 안내 |
| `aggregate` | 인구통계 컬럼 GROUP BY COUNT |
| `sample_persona` | 필터 조건부 무작위 샘플 |
| `search_persona` | FTS5(trigram) 자유 텍스트 + 필터 검색 |
| `get_persona` | uuid로 전체 컬럼 조회 |

검색어는 trigram 특성상 **3글자 이상**이어야 하며, 조사가 붙은 형태도 부분 문자열로
매칭된다 (`등산과`, `트로트`). prefix `*`는 불필요하다.

## 사례 1 — 합성 설문 / FGD 시뮬레이션

> 동작하는 데모: [`examples/synthetic_survey.py`](../examples/synthetic_survey.py)

### 시나리오

신규 구독 요금제, 정책 안내문, 광고 카피를 출시 전에 "인구 비례 가상 패널"에게
미리 보여주고 반응을 수집한다. 실제 설문 대비 비용이 수백 분의 일이고 즉시 반복 가능하다.

### 도구 호출 흐름

```jsonc
// 1) 모집단 분포 확인
aggregate({"group_by": ["province", "sex"]})

// 2) 분포 비례로 층화 샘플링 (셀별 반복 호출)
sample_persona({"filters": {"province": "경기도", "sex": "여자"}, "n": 3, "full": true})

// 3) 각 페르소나로 롤플레이 → 응답 수집 → 가중 집계
```

### 프롬프트 설계

- **시스템 프롬프트**: 페르소나 전체 컬럼(서술문 10종 + 인구통계)을 그대로 넣고
  "당신은 아래 인물이다. 이 인물의 가치관·생활환경에서 벗어나지 말 것"을 지시.
- **응답 형식 고정**: 5점 척도 + 한 줄 이유를 JSON으로 강제해 집계 가능하게 한다.
- **온도**: 다양성이 목적이므로 낮추지 않는다 (기본값 권장).

### 한계 / 주의점

- **실제 여론조사를 대체하지 않는다.** LLM의 사전학습 편향이 응답에 섞이므로
  방향성 탐색·문안 비교용으로만 사용하고, 의사결정 근거로는 실측을 병행할 것.
- 표본 수가 작으면 셀별 분산이 크다. 축별 비교가 목적이면 셀당 최소 5명 이상 권장.

## 사례 2 — 챗봇/에이전트 평가용 유저 시뮬레이터

### 시나리오

CS 챗봇·상담 에이전트를 배포 전에 다양한 가상 고객으로 두들겨본다.
연령·지역·직업이 다른 사용자가 만드는 어휘 차이("환불 규정" vs "돈 돌려주는 거"),
요구사항 차이가 엣지케이스를 자동으로 발굴한다.

### 도구 호출 흐름

```jsonc
// 평가 축을 정해 페르소나 패널 구성 (예: 연령대별 5명씩)
sample_persona({"filters": {"age_min": 60}, "n": 5, "full": true})
sample_persona({"filters": {"age_max": 29}, "n": 5, "full": true})
```

이후 에이전트 루프: 페르소나 LLM(고객 역) ↔ 평가 대상 챗봇이 멀티턴 대화
→ 별도 LLM-as-judge가 "과제 해결 여부 / 만족도 / 실패 지점"을 채점.

### 프롬프트 설계

- 고객 역 시스템 프롬프트에 페르소나 + **구체적 과제**(예: "요금제를 해지하고 싶다.
  단, 위약금이 있으면 망설인다")를 부여. 과제는 페르소나의 `occupation`,
  `family_persona` 등에서 파생시키면 자연스럽다.
- judge에는 대화 전문과 평가 루브릭만 주고 페르소나 원문은 주지 않는다
  (고객 정보를 모르는 상태에서 챗봇 응대 품질만 평가).

### 한계 / 주의점

- 인구통계 축별 성공률 차이는 **챗봇 편향의 신호**일 수도, 시뮬레이터 자체 편향일
  수도 있다. 축별 비교 시 동일 과제를 고정하고 페르소나만 바꿔 통제할 것.

## 사례 3 — 한국어 합성 데이터 생성 (SDG)

### 시나리오

파인튜닝·RAG 평가용 한국어 데이터셋(상품 리뷰, 상담 로그, 커뮤니티 게시글 등)을
페르소나 시드로 생성한다. Nemotron-Personas 데이터셋의 원래 설계 목적이기도 하다.
시드가 1,000,000명이라 생성물의 다양성 붕괴(mode collapse)를 막기 좋다.

### 도구 호출 흐름

```jsonc
// 도메인에 맞는 시드 풀 구성 — 검색 또는 무작위
search_persona({"query": "캠핑", "fields": ["hobbies_and_interests"], "limit": 100})
sample_persona({"n": 100, "full": true})
```

페르소나마다 "이 사람이 쓸 법한 캠핑용품 리뷰 3건"처럼 생성을 지시하고,
인구통계 컬럼을 생성물의 **메타데이터 라벨**로 함께 저장한다
(나중에 축별 균형 재조정·필터링에 사용).

### 한계 / 주의점

- 원본 데이터셋 라이선스는 **CC BY 4.0** — 파생 데이터셋 공개 시 출처 표기 필요.
- 페르소나 서술문을 생성물에 그대로 복사하지 않도록 "서술문 표현을 재사용하지 말 것"을
  프롬프트에 명시 (n-gram 중복 검사 병행 권장).

## 사례 4 — 마케팅 세그먼트 테스트 / 캐릭터 캐스팅

### 시나리오 A: 카피 A/B 테스트

trigram 검색으로 관심사 기반 세그먼트를 정의하고, 해당 세그먼트 페르소나들에게
카피 A/B를 평가시킨다.

```jsonc
// "등산도 하고 트로트도 듣는" 세그먼트
search_persona({
  "query": "등산과 AND 트로트",
  "filters": {"age_min": 50, "sex": "여자"},
  "limit": 30, "full": true
})
```

각 페르소나에게 두 카피를 무작위 순서로 제시하고 선호 + 이유를 수집 → 선호율 비교.
사례 1과 같은 집계 구조이므로 `synthetic_survey.py`를 거의 그대로 변형할 수 있다.

### 시나리오 B: 캐릭터 / NPC 캐스팅

소설·시나리오·게임에서 배경이 내적으로 일관된(직업·가족·지역·취미가 서로 맞물리는)
인물이 필요할 때 즉시 후보를 공급한다.

```jsonc
// "부산 영도구 50대 자영업자" 후보 5명
sample_persona({
  "filters": {"province": "부산광역시", "district_like": "영도%",
              "age_min": 50, "age_max": 59, "occupation_like": "%자영업%"},
  "n": 5, "full": true
})
```

LLM은 후보들의 서술문을 바탕으로 대사 톤·말버릇·갈등 요소를 파생시킨다.
조연 수십 명이 필요한 장편·게임에서 특히 유용하다.

### 한계 / 주의점

- 페르소나는 실존 인물이 아니지만 실제 분포를 모사하므로, 특정 지역·직업 집단을
  희화화하는 용도로 쓰지 않도록 생성 가이드라인을 함께 운영할 것.
